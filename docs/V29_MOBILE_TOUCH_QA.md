# V29 移动端体验与触控优化 QA

## 版本目标

在 V28「第一案全流程 QA + 新手引导优化」基础上，专门优化手机端和窄屏体验，让玩家在手机上也能顺畅完成第一案：点击热点、人物互动、查看证据、完成轻解谜、回律所推理、进入法庭翻盘。

---

## 本版优化内容

### 1. 移动端视口稳定

新增 `updateMobileViewportUnit()`：

- 设置 `--app-vh`，减轻移动浏览器地址栏导致的高度跳动
- 根据 `pointer: coarse` 自动给 `body` 增加 `touch-device` class
- 监听 `resize` 与 `orientationchange`

### 2. 触控目标优化

新增统一触控规范：

- 关键按钮、选项、卡片、热点、人物按钮设置触控优化
- 粗指针设备下关键点击目标不小于 44px
- 场景热点在移动端扩大到 54px 至 62px
- 禁止核心交互只依赖 hover

### 3. 首页与菜单移动端优化

优化：

- 首页主标题尺寸
- 主菜单按钮单列布局
- 按钮高度和可点击区域
- 顶部工具按钮换行与触控尺寸

### 4. 场景与热点优化

优化：

- 竖屏场景高度：`clamp(360px, 64vh, 620px)`
- 小屏下热点更大、更容易点击
- 热点标签允许换行，避免挤出屏幕
- 镜头模式下背景缩放更适合窄屏

### 5. 人物舞台触控优化

优化：

- 移动端人物点击区域更稳定
- 人物徽章在极窄屏下隐藏，避免遮挡
- 角色卡片高度和半身站位更适合手机竖屏
- 人物仍然是 button，可点击、可聚焦

### 6. 对话框与文字可读性优化

优化：

- 手机端隐藏头像，释放文字空间
- 对话框改为单列布局
- 文字字号和行高适配手机阅读
- 对话文本区域可滚动
- 选项单列排列，点击目标更大

### 7. 证据包与任务卡优化

优化：

- 证据包增加移动端滚动体验
- 任务 / 导演流程 / 法庭高潮 / FIRST RUN GUIDE 卡片可读性优化
- 阶段列表在手机端限制高度并可滚动

### 8. 轻解谜与人物交互优化

优化：

- 谜题卡片移动端单列布局
- 交互卡片、证据出示卡片单列
- 人物交互头像使用 contain，避免裁切过狠
- 弹窗内容区域可滚动

### 9. 物件特写与 CG 移动端优化

优化：

- 物件特写弹窗移动端单列
- 操作按钮全宽
- CG 画面在竖屏下提高高度
- CG 标题和旁白字号适配
- 横屏低高度设备单独适配
- 保留原有高质量图像，不改变核心资源路径
