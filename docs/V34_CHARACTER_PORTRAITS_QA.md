# V34 Character Portraits & Expressions QA

## 版本目标

V34 对上一轮评分中“人物演出感”短板进行第一阶段优化。重点不是新增剧情系统，而是补齐主要角色的阶段性表情立绘资源，让对白框、人物舞台、人物交互和 CG 中的人物更像在表演，而不是只靠 CSS 亮暗变化。

---

## 本版新增 / 优化内容

### 1. 新增主要角色表情立绘资源

新增目录：

```text
assets/portraits/
```

新增 19 张阶段性表情立绘 PNG，尺寸统一为 `1024 × 1536`。

覆盖角色：

- 陆沉：neutral / serious / resolve
- 韩亦：neutral / serious
- 苏岚：neutral / sad / shock
- 陈巍：anger / shock / sad
- 李娜：sad / shock
- 莫野：neutral / serious
- 乔衡：serious / anger / shock / sad

说明：陈巍、李娜、乔衡等角色没有单独 neutral 原图时，默认头像会映射到最适合的可用情绪版本。

### 2. 新增表情立绘调用系统

新增：

- `actorExpressionPortraits`
- `portraitFor(name, emotion)`
- `showExpressionGallery(name)`

现在以下系统会根据角色情绪调用对应立绘：

- 对话框 speaker portrait
- 人物舞台
- 人物交互面板
- CG / 动态插图角色

### 3. 人物交互新增“表情资源”入口

点击场景人物后，人物交互菜单新增：

```text
表情资源
```

可查看当前角色已有表情立绘资源。

### 4. 默认角色头像更新

`data/game-data.json` 中主要角色默认 `portrait` 已指向 V34 新表情资源的 neutral / 默认版本。

## 说明与限制

本版是“阶段性表情资源”版本。由于没有独立绘师手绘多套差分，部分表情仍基于现有角色图进行色调、光效和情绪化处理；它能明显提升人物演出反馈，但还不能完全替代真正的商业级多表情立绘。

下一步如继续提升人物演出，建议针对陆沉、苏岚、陈巍、乔衡四名第一案核心角色，生成或绘制真正的多表情半身立绘。
