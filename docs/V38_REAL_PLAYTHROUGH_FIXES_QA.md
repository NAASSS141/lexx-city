# V38 第一案全流程真人试玩修复版 QA

## 版本目标

V38 基于 V37 第一案镜头剪辑与台词节奏精修版，根据本轮真实试玩反馈修复界面卡点。目标不是新增大系统，而是减少上下滑动、降低右侧信息密度，并修复地图 / 图鉴 / 时间线等系统面板没有以弹窗覆盖显示而是掉到页面底部的问题。

---

## 收到的问题

### 1. 右侧排版过于详细

反馈：右侧区域信息太多，保留必要选项即可。

### 2. 地图等系统面板位置不对

反馈：点击地图等功能时，没有像弹窗一样覆盖显示，而是呈现在页面最下方，需要一直上下滑动页面。

### 3. 减少上下滑动次数，适配屏幕

反馈：希望整体布局更贴合屏幕，不要频繁上下滑。

---

## 逐条修复项

### 1. 右侧任务区改为紧凑模式

修改 `renderTaskTracker()`：

- 不再默认展开完整任务阶段列表
- 只显示当前目标、进度和必要按钮
- 增加 `任务详情` 按钮，详细任务仍可打开查看
- 保留 FIRST RUN GUIDE、导演流程、法庭高潮，但全部改为紧凑卡片

新增：

- `renderCase1QuickAction()`

它会根据当前状态优先提供一个主要行动按钮。

### 2. 导演流程 / 法庭高潮卡片压缩

修改：

- `renderCase1DirectorCard()`
- `renderCase1CourtClimaxCard()`
- `renderCase1GuidanceCard()`
- `renderDirectingBeatCard()`

清理内容：

- 移除右侧长列表
- 保留当前节点、进度和主按钮
- 详细内容交给弹窗面板处理

### 3. 修复系统面板掉到页面底部的问题

根因：后续样式中 `#app,.modal,.toast-stack,.chapter-splash` 这组规则把 `.modal` 的 `position` 覆盖成了 `relative`，导致地图、图鉴、任务等弹窗不再固定覆盖屏幕，而是在文档流里排到页面底部。

修复：

- V38 追加 `.modal{ position:fixed !important; inset:0 !important; z-index:9999 !important; }`
- 保证地图、图鉴、时间线、任务、存档、设置等系统面板都以覆盖弹窗显示
- 弹窗打开时锁定页面滚动
- 弹窗关闭时恢复页面滚动

新增：

- `lockPageForModal()`
- `unlockPageForModal()`

### 4. 减少上下滑动

布局优化：

- 右侧栏桌面端 sticky
- 右侧栏内部可滚动
- 证据包降低单条高度
- 隐藏右侧证据说明文字，只保留名称和图标
- 场景行动按钮隐藏次要说明，只保留必要操作
- 地图弹窗内部滚动，不再拖长整个页面
- 移动端保留可滚动弹窗，但不再把内容推到页面底部
