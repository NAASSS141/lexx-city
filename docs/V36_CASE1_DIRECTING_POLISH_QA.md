## 镜头 / 台词 / 人物状态调整

### 1. 新增第一案镜头调度表

新增：

- `case1DirectingCues`
- `applyCase1DirectingCue(id, opts)`
- `cueForEvidence(name)`
- `applyCase1SceneEntryCue(sceneId)`

根据当前场景或获得的证据触发不同镜头类型：

- `wide`：开场 / 法庭建立镜头
- `medium`：人物对峙
- `close`：人物状态变化
- `insert`：证据 / 物件特写
- `impact`：法庭击穿、乔衡破防
- `black`：第一案结局收束

### 2. 人物戏剧状态联动

第一案核心人物会随节点进入更准确状态：

- 陆沉：`observe / counter / pressure`
- 苏岚：`fear / waver / confess`
- 陈巍：`wronged / breakdown`
- 乔衡：`calm / break / evade`

触发来源包括：

- 场景进入
- 证据获得
- 组合推理完成
- 轻解谜完成
- 关键对白
- 法庭高潮流程

### 3. 台词节奏处理

新增：

- `lineBreakKeyPhrases(text)`
- `directingCueForDialogue(name, text)`

对第一案重点台词做分行处理，让阅读节奏更接近视觉小说对白。例如：

- 陈巍否认刺人
- 苏岚说明无法报警
- 苏岚交出未发送语音

### 4. 轻解谜与证据链镜头联动

新增：

- `directingCueForPuzzle(id)`

轻解谜完成后会触发对应镜头和人物状态：

- `c1_timeline_order` → 律所证据墙 / 组合推理
- `c1_contradiction_sulan` → 苏岚坦白
- `c1_submit_blackcar` → 关键证据提交

### 5. 导演节拍卡

新增：

- `currentDirectingBeat()`
- `renderDirectingBeatCard()`

任务区会显示当前第一案导演节拍，例如：

- 雨夜路口 / 商业区调查
- 便利店现场 / 苏岚询问
- 陈巍会见对峙
- 律所证据墙推理
- 第一案法庭高潮
- 第一案结局收束
