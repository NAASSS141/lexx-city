# V36 Text Cleanup QA

## 版本目标

根据反馈，删除或改写过度解释、过度感悟、替玩家解释证据作用的文案。

用户举例要求删除的类型包括：

- `时间线出现裂缝。`
- `投影幕被点亮的瞬间，庭审不再只是口供的较量。光束把真相的一角，钉在每个人面前。`

本版按这个方向排查并清理了同类情况。

---

## 本版修改内容

### 1. CG / 动态插图文案清理

处理方式：

- 清空 CG 解释性 `narration`
- 保留标题
- 保留简短 `caption` 场景标注

示例：

- `关键证据提交`
- `第一案法庭`
- `便利店现场`
- `律所证据墙`
- `第二案线索`

同时修改 `showCgIllustration()`，当 `narration` 为空时隐藏对应段落，避免空白说明区域占位。

### 2. 证据描述清理

`evidenceMeta` 中的说明改为中性标签，不再解释证据作用。

示例：

- `22:13 小票`：`付款小票`
- `破损监控记录`：`监控记录`
- `黑车车牌残片`：`车牌残片`
- `未发送语音`：`手机语音`

### 3. 场景描述清理

主要场景描述改为地点 / 物件标注，不再加入感悟式描写。

示例：

- 商业区路口：`商业区路口。警戒线、监控探头、斑马线与排水沟。`
- 第一案法庭：`第一案法庭。进行终案推理与证据提交。`
- 律所办公室：`律所办公室。证据墙、案件笔记与城市地图。`

### 4. 第一案流程提示清理

清理范围：

- FIRST RUN GUIDE
- 第一案导演流程
- 第一案法庭高潮流程
- 轻解谜提示与成功文案
- 部分 Toast / 任务提示

处理原则：

- 保留操作指向
- 删除感悟式总结
- 删除“替玩家解释证据意义”的句子
- 避免“裂缝 / 光束 / 真相的一角 / 替罪羊”等抽象表达

### 5. 人物互动文案清理

部分人物观察和证据反馈改为中性描述。

例如：

- 陆沉观察：`陆沉正在核对证据与时间顺序。`
- 陈巍观察：`陈巍情绪激动，反复否认自己刺人。`
- 乔衡观察：`乔衡回避关键问题，保持沉默。`

### 6. 结局与推理文案清理

保留结局逻辑，不改结局判定。

修改方向：

- 减少“感悟式总结”
- 保留事实结果
- 保留下一案入口
