# V14 场景探索系统 QA

本阶段新增场景图热点探索，不删除原有行动按钮，不修改剧情数据和证据判定逻辑。

## 新增能力

- `sceneHotspots`：每个主要场景的热点配置
- `renderHotspots()`：根据当前场景渲染可调查热点
- `handleAction()`：将原有行动按钮与场景热点统一到同一套行动处理逻辑
- 热点状态：
  - `CHECK` 调查
  - `TALK` 对话
  - `MOVE` 移动
  - `OPEN` 打开系统面板
  - `LOCKED` 未解锁
  - `DONE` 已调查

## 保留项

- 原有行动按钮仍然存在
- 原有证据获取逻辑保持不变
- 原有对话、结局、成就、存档逻辑保持不变
- 移动端可通过点击热点操作，不依赖拖拽
